Key findings

  • 5,487 smartwatch water temperature observations were analysed.
  • These observations formed 261 swim sessions using a 30-minute gap rule.
  • 115 sessions (44.1%) contained five or more readings and provide sufficient observations to investigate stable temperature plateau detection.
  • The median sampling interval was 17.0 seconds, providing high temporal resolution within sessions.
  • The data support deriving one representative temperature per swim session, rather than treating observations as independent temperature measurements.

Initial interpretation

  • The smartwatch data are best interpreted at swim-session level, not individual-reading level.
  • The data should not be treated as a complete diary of all swims.
  • Recording behaviour may differ between devices or software versions.
  • High values suggest some readings are affected by body heat or post-swim warming.
  • Produce a validated smartwatch_sessions.csv dataset containing one representative temperature per swim session.

Methodological implications

This report demonstrates that smartwatch water temperature observations are best interpreted as swim sessions rather than independent temperature measurements.

  • Repeated measurements within a swim session provide information about sensor equilibration behaviour.
  • Representative water temperatures should therefore be derived at the session level rather than from individual observations.
  • High temperature observations are consistent with body heat or post-swim warming and should not automatically be interpreted as seawater temperature.
  • The representative water temperature for each swim session will be derived from a stable temperature plateau following sensor equilibration.
  • Thresholds used within the stable plateau algorithm will be selected using evidence from this sensor characterisation rather than predetermined values.

Questions under investigation

The purpose of this report is to characterise smartwatch behaviour as a water temperature sensor and to provide evidence for a reproducible processing methodology.

  1. How quickly does the smartwatch sensor equilibrate with surrounding seawater?
  2. How stable are temperature observations once thermal equilibrium has been reached?
  3. How many observations are required to estimate a representative session temperature reliably?
  4. Which stable plateau detection method provides the most robust and reproducible representative water temperature?
  5. How closely do representative smartwatch-derived temperatures agree with independent coastal sea temperature observations?

Next steps

  2. Quantify time to thermal equilibrium.
  3. Define an evidence-based stable plateau algorithm.
  4. Validate representative temperatures against independent observations.
  5. Publish a reproducible session-level dataset.
